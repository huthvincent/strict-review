## 一、死因解剖（四个凶手，按危害排序）

### 主凶 A：leg3 被从"独立的腿"降级成"标签装饰"

- v1 的 leg3 是**独立发现生成器**：轻量分类器把 commit 归到叶子后，**自己发出**
  一条 risk-route finding（置信门 0，全路由）——它承载了 v1 几乎全部召回
  （消融 −15.1pp），在 61 个前瞻 commit 上触发 43%。
- v2 实现里（`detector_v2.py`）：`_route_leaves()` 的分类结果**只用来裁剪手册页**；
  `_attach_benchmark()` 只给**主审已产出的** finding 贴 `suggested_benchmark` 标签。
  **主审不开口，leg3 就永远沉默。** 独立路由发现在 v2 里根本不存在。
- 这直接解释 pair@2 = 0%、前瞻 0/61（v1 同批 24/61，severe 口径）。
- 责任：作业书 3.4 的措辞"leg3 路由保留：分类→叶子→查表，**finding 附**
  suggested_benchmark"有歧义——"附"字诱导了装饰式实现。**主 agent 起草责任。**

### 主凶 B：三问纪律被实现成 severity 硬闸

- `SYS_MAIN` 原文："**只有三问都指向真实退化才报 important/critical；否则不报或
  降为 suggestion**"。而计分规则只认 critical/important。
- 后果：全轮 19 条 finding 里 **15 条被降级为 suggestion**（计分上不存在），
  仅 4 条 important、0 critical——recall 数学上限被锁死在 4/150≈2.7%。
- 铁证：前瞻里 v2 对 `bcf4c8fb`（默认路由策略三处齐改）给出 conf 0.72 的
  finding——内容正确，却按规则只敢标 suggestion。**它看见了，但被规则捂住了嘴。**
- 责任：作业书 3.3"宁缺毋滥"由主 agent 起草，执行机把它形式化成硬闸——共同责任。

### 从犯 C：常驻手册的"白名单效应"

- 消融：去手册 recall 2.0%→8.0%（原始触发 10%→16%，方向一致；CI 重叠，
  且判后口径离线不可复现——见验收遗留 3）。
- 机制推断：手册页作为"先验"注入后，模型把"不匹配任何已知反模式"当成
  "大概率良性"的证据——参考资料变成了准入清单。

### 从犯 D：省钱补丁砍断了大 commit 的核验能力

- 冒烟发现大 commit 分解 $1.83/item 太贵 → 改 `decomp_max_turns=2`＋`groups≤6`。
- 但三问纪律**要求**用父快照工具核实调用方——2 轮内（约 1 次工具调用后即被强制
  emit）根本做不完核验 → 按主凶 B 的规则只能降级或沉默。
- 引入侧 diff 中位数大 8.6×，dev pair 57% 触发分解——**受害最重的恰是守门员
  真正的战场**。groups≤6 还静默丢弃第 7 组之后的内容（宪章 2.3 违例）。

### 流程级根因：冻结门没有"活性检查"

- **验收铁证（S3 fail）**：冒烟集 25 项只有 2 项出 finding（1 important）——
  哑火在冻结前清晰可见；但成本修复后的复跑只核了"成本降没降、leak 是否 0"，
  没有任何报告披露冒烟命中率，随后照常冻结。
- Stage 4 的崩塌本可在这里被 $14 的成本拦下。

## 二、判分系统无罪，但"充分验证"言过其实

- 真证据：S0 锚检——30 条 RUN2 非空预测复判，一致率 96.7%、κ=0.902。
- 伪证据：κ=1.000（52 样本中约 91% 是空 finding 的平凡一致，pos×HIT 层实际只有
  2 项，报告写 4 项有误）；Nova 100%（22 样本中 20 个空 finding）。
  正确表述应为"复检因 finding 极稀疏而信息量有限"。

- **全绿**：cache-only 落地、T1/T4 重建 0 miss、McNemar×5 且预登记规则被诚实执行

  误杀 0.09%、520 行预测 leak=0、判据宣判如实、回传清单齐（含 judge 缓存与 paper/）。
- **数字级问题（4 条 fail，均不动摇主结论）**：①κ/Nova"充分验证"表述（见上）；

## 五、v2.1 处方（外科手术，不是推倒重来）

1. **复活 leg3 为独立腿**：把 v1 的 leg3 代码原样并回，其输出与主审 finding 做
   并集合并再进 budget 排序。仅此一刀即可恢复 ≈v1 的召回地板。
2. **三问改为置信度校准器**：答不全 → 降 confidence 并在 finding 里注明哪问未核，
   **不降 severity、不禁止上报**。
3. **手册从常驻先验改为按需工具**：`lookup_handbook(leaf)` 由模型主动调用；
   prompt 明示"手册未覆盖不代表良性"。
4. **大 commit 两级审**：便宜的分组初筛排序 → 只对 top-2 可疑组恢复 5 轮深审；
   丢弃组数必须写进 meta（宪章 2.3）。
5. **冻结门加活性判据**（已入宪章 7.6）：冒烟集已知正样本上的 severe-finding 率
   低于下限（建议 ≥25%）→ 禁止冻结。

预期：1+2 两刀合计即应回到 v1 水平之上（v2 的 FPR 0% 留出了大量放宽空间——
油门焊死的车，方向盘并没有坏）。
